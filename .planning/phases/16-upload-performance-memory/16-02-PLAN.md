Replace the entire body of `uploadAttachments()` after `splitIntoBatches(fileInfos)` with the parallel implementation from the RESEARCH.md Pattern 1. Key structure:

a) Compute `parallelism = Math.min(batches.size(), this.config.testops.batch.uploadThreads)`
b) Create `ExecutorService pool = Executors.newFixedThreadPool(parallelism)`
c) Save original read/write timeouts
d) Compute max timeout across all batches, apply once before submitting callables
e) Create `AttachmentsApi api = createAttachmentsApi()` — single instance shared across callables (OkHttp connection pool is thread-safe; timeout is set once, not per-callable)
f) Build `List<Callable<List<String>>>` where each callable calls `uploadBatchParallel(api, batch, batchIdx, totalBatches)`
g) `pool.invokeAll(callables)` then collect results from futures
h) `finally` block: `pool.shutdownNow()`, restore original timeouts

Add private helper `uploadBatchParallel(AttachmentsApi api, List<FileInfo> batch, int batchIdx, int totalBatches)`:

Handle `InterruptedException` from `invokeAll()`: re-interrupt thread, log warn, return empty list.
