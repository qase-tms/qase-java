<objective>
Disk-backed attachment staging with immediate memory release in ApiClientV1.prepareFiles()

Purpose: Eliminate heap memory pressure from large attachments. Currently, byte[] content stays live in heap from addResult() through the entire async upload pipeline. This change writes content to temp files immediately, nulls out the byte[] reference so GC can reclaim it, and uses File.createTempFile() for collision-safe unique paths (critical for the parallel uploads coming in Plan 02).

Output: Modified prepareFiles() with temp file creation, a sanitizeForFileName() helper, and TempFileAttachmentTest with tests covering null-out, temp dir location, and cleanup.
</objective>
